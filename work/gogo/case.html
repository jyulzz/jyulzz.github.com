Context

GOGO1 is a well-established travel brand specialized in summer holidays for 16 to 25 year-olds in Southern Europe destinations.

GOGO travellers can use the GOGO application before, while and after being on location, to see information about their holiday, contact customer service, find hotspots in their area, and more.

The application's most popular section is a Flirting section where users can find each other, flirt and chat.

Users of the application have the ability to give feedback, including feature requests.

Summary

Role

  • Curation of feature requests from users,
  • Refinement, design, implementation and testing of new feature.

Deliverable

User flows and UI designs for the new picture management feature within the profile section of the application.

Team

  • 10 developers
  • 1 designer (me)
  • 1 product manager

Results

  • New flows were successfully implemented in version 1.8,
  • Highly positive user feedback on the new feature and increase in overall application usage.

Process

Research

Out of 81 feature requests received, 65 asked to be able to include more than 1 picture on user profiles.

It was subsequently decided to add this new feature in the next release.

Planning

Changes to the user flow were clarified in collaboration with the team of developers during 2 refinement sessions.

Production

I designed iOS and Android versions of the 9 new screens and used them into an updated flowchart :

As the main UI control for this updated profile picture user flow, I opted for a matrix interface, because it is a familiar pattern in this use-case, for instance Tinder uses a similar UI component for the same purpose. It allows for simple tap actions to add, edit or delete pictures, it clearly shows the order of each picture within the gallery, giving a complete overview of the gallery at every step of the flow :

I designed an accessory flow for a user to edit their profile pictures from their profile:

I added a swipping control —the dots above the name tag— to allow users to swipe within the area of a profile picture to see other pictures from the same gallery:

The last major change I introduced was the use of native interfaces provided by iOS and Android wherever possible, such as selecting pictures from their phones' camera roll, as they are familiar to users, and well-integrated with the OS. This decision also freed up development resources that were subsequently directed to producing features unique to the application while reducing legacy and update tasks as well in the future.

Testing

Pre-release:

  • Guerilla testing by the team,
  • User observations.
Post-release:
  • Feedback messages and application store rating monitoring.

Final Thoughts

Young travelers expect first class digital services. For a brand like GOGO, the fast deployment of new features, including requested ones, is an important condition to maintain growth in the number of active users. Achieving this change in a critical flow within the most popular section of the application in a short period of time was an interesting challenge.
